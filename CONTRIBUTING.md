# Contributing

Thanks for your interest! Here's how to submit a tool or skill.

## Adding a new entry

1. Fork this repository
2. Add your entry to the appropriate section in `README.md`
3. Follow the format: `| [Name](url) | ![Stars](badge) | One-line description |`
4. Submit a pull request

## Star badge format
```
![Stars](https://img.shields.io/github/stars/OWNER/REPO?style=flat-square)
```

## Quality criteria

- **Working code** — must have a functional demo or documented usage
- **Open source** — MIT, Apache 2.0, or similar permissive license
- **Maintained** — commit activity within last 3 months (exceptions for stable, complete tools)
- **Documented** — clear README with installation and usage instructions
- **Novel** — brings something new vs existing entries

## What we don't accept

- Paid products (even with free tiers)
- Forks with no meaningful changes
- Abandoned projects (no commits in 6+ months, unless complete/stable)
- Self-promotion without substance

## Suggesting a category

Open an issue with the `category` label describing what's missing and why it matters.

## Reporting issues

If a listed tool is broken, abandoned, or misrepresented, open an issue with the `report` label.
